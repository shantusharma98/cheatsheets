CardinalityMeaningExample

What to spot in this formula: =VLOOKUP(A2, Sheet2!B:D, 2, 0)

• A2 = lookup value (Employee ID from this sheet)
• Sheet2!B:D = lookup range on Sheet2 — BUT this is a problem! If the Employee ID being looked up is in column B of Sheet2, and the range starts at B, then column B = position 1, C = position 2.
• 2 = return the 2nd column of the range (column C of Sheet2)
• 0 / FALSE = exact match ✓

Why row 3 shows #N/A: Employee E1043 was not found in Sheet2!B:D — either it doesn't exist there, or it has a type mismatch (text vs number) or trailing spaces.
Common #N/A causes: 1) Value genuinely not in lookup range. 2) ID stored as text in one place, number in another. 3) Trailing space " E1043" ≠ "E1043". Fix with TRIM() and VALUE().

Read this query — what does it output? Identify every clause and what each does.
Query Reading · Core skill for all SQL interviewsMUST KNOW

Read and explain this query
SELECT c.CompanyName, COUNT(o.OrderID) AS TotalOrders, ROUND(AVG(od.UnitPrice),2) AS AvgPrice, SUM(od.Quantity) AS TotalQty FROM Customers c JOIN Orders o ON c.CustomerID = o.CustomerID JOIN OrderDetails od ON o.OrderID = od.OrderID WHERE c.Country = 'Germany' GROUP BY c.CompanyName HAVING COUNT(o.OrderID) > 5 ORDER BY TotalOrders DESC LIMIT 10;
ClauseWhat it does
SELECTReturns 4 columns: company name, count of orders, average unit price, total quantity
FROM + 2 JOINsConnects 3 tables: Customers → Orders → OrderDetails (standard Northwind chain)
WHERE Country='Germany'Filters ROWS before grouping — only includes German customers
GROUP BY CompanyNameCollapses all rows per company into one summary row
HAVING COUNT > 5Filters GROUPS after aggregation — only companies with more than 5 orders
ORDER BY TotalOrders DESCSorts results — highest order count first
LIMIT 10Returns only top 10 rows
Plain English: "Show me the top 10 German customers by order volume, but only show those with more than 5 orders, including their average unit price and total quantity ordered."
S2
Identify the error in this SQL query — it runs but gives wrong results. What's the bug?
Query Debugging · Silent errors are the hardestMUST KNOW
▼
Find the silent bug
SELECT c.CustomerName, SUM(o.Revenue) AS TotalRevenue FROM Customers c LEFT JOIN Orders o ON c.CustomerID = o.CustomerID WHERE o.Status = 'Completed' ← BUG IS HERE GROUP BY c.CustomerName;
Bug: The WHERE filter on the right table (o.Status) converts the LEFT JOIN into an INNER JOIN — silently! Customers with no completed orders will have o.Status = NULL after the LEFT JOIN, and NULL = 'Completed' evaluates to FALSE (not NULL), removing those rows.

Intent was: Show ALL customers, even those with no completed orders (show NULL for them).
Actual result: Only shows customers who have at least one completed order — same as INNER JOIN.

Fix — move filter into ON clause:
SELECT c.CustomerName, SUM(o.Revenue) AS TotalRevenue FROM Customers c LEFT JOIN Orders o ON c.CustomerID = o.CustomerID AND o.Status = 'Completed' ← filter in ON preserves NULL rows GROUP BY c.CustomerName;

Identify the error in this Pandas code — what warning or wrong result will it produce?
SettingWithCopyWarning · Classic pandas trapMUST KNOW
▼
Find the bug
# BUG CODE: it_df = df[df['Dept'] == 'IT'] it_df['Bonus'] = it_df['Score'] * 0.1 # ⚠️ SettingWithCopyWarning! # df['Bonus'] might NOT be updated # FIX 1: use .loc on original df df.loc[df['Dept'] == 'IT', 'Bonus'] = df.loc[df['Dept'] == 'IT', 'Score'] * 0.1 # FIX 2: explicit copy if you want a separate df it_df = df[df['Dept'] == 'IT'].copy() it_df['Bonus'] = it_df['Score'] * 0.1 # safe — modifying a true copy
Why it happens: Boolean indexing df[condition] may return a view OR a copy depending on pandas internals — you can't predict which. Assigning to a view modifies the original. Assigning to a copy does nothing to the original. The warning tells you "I can't guarantee which one this is."

What is the output of this code? Explain why — many candidates get this wrong.
Python Mutability · Classic interview trapMUST KNOW
▼
What prints?
a = [1, 2, 3] b = a b.append(4) print(a) # ? print(b) # ? print(a is b) # ? c = a.copy() c.append(5) print(a) # ? print(c) # ?
print(a) → [1, 2, 3, 4] # changed! b and a point to same list
print(b) → [1, 2, 3, 4] # same object
print(a is b) → True # same memory address
print(a) → [1, 2, 3, 4] # .copy() made new object — a unchanged
print(c) → [1, 2, 3, 4, 5] # c has the 5, a doesn't
Key rule: In Python, = assigns a reference, NOT a copy. For lists, dicts, DataFrames — always use .copy() when you need an independent copy.

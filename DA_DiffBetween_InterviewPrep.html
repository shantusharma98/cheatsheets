Second highest salary — the 3 correct methods (direct from your feedback)
Classic Interview ProblemMUST KNOW
▼
-- Method 1: Subquery with MAX (simplest, often asked first)
SELECT MAX(Salary) FROM Employees
WHERE Salary < (SELECT MAX(Salary) FROM Employees);

-- Method 2: DENSE_RANK (safest — handles ties correctly)
SELECT Salary FROM (
  SELECT Salary, DENSE_RANK() OVER (ORDER BY Salary DESC) dr
  FROM Employees
) t WHERE dr = 2;

-- Method 3: LIMIT/OFFSET (MySQL/PostgreSQL)
SELECT DISTINCT Salary FROM Employees
ORDER BY Salary DESC LIMIT 1 OFFSET 1;
Trap: Method 1 fails with ties. If two people share the MAX salary, it returns the second person's salary which is actually the HIGHEST. Always use DENSE_RANK for correctness.

import numpy as np

a = np.array([[1,2,3],
              [4,5,6]])   # shape: (2, 3) → 2 rows, 3 columns

# Indexing: [row, col]
a[0, 1]       # → 2 (row 0, col 1)

▼

What it is: A symmetric bell-shaped distribution where most values cluster around the mean. Defined by two parameters: mean (μ) = center, std dev (σ) = spread.

Hypothesis testing — step-by-step structure (from your feedback)
Inferential Statistics · Direct from your feedbackMUST KNOW
▼
StepActionExample (testing if new drug works)
1State H₀ and H₁H₀: Drug has no effect (mean = 0). H₁: Drug has effect (mean ≠ 0)
2Choose significance level αα = 0.05 (5% chance we reject H₀ when it's actually true)
3Choose the right testt-test (small sample, unknown population std dev)
4Calculate test statistic + p-valueRun ttest, get p = 0.03
5Compare p-value to α0.03 < 0.05 → Reject H₀
6Interpret in context"Statistically significant evidence that the drug has an effect at 5% level"
p-value misinterpretation trap: p=0.03 does NOT mean "97% chance the drug works." It means "IF the drug had no effect, there's only a 3% chance of seeing this result by random chance." The actual probability of H₁ being true requires Bayesian reasoning.
Type I error (α): Rejecting H₀ when it's true = false positive. Type II error (β): Failing to reject H₀ when H₁ is true = false negative. Power = 1 - β.
ST4
Type I vs Type II error — with a real-world analogy
Hypothesis TestingHIGH PRIORITY
▼
Error TypeDefinitionFire Alarm Analogy
Type I (α) False PositiveRejecting H₀ when H₀ is actually TRUE. Claiming an effect exists when it doesn't.Fire alarm goes off when there's NO fire — false alarm. Scary but usually low consequence.
Type II (β) False NegativeFailing to reject H₀ when H₁ is TRUE. Missing a real effect.Fire alarm DOESN'T go off when there IS a fire — missed detection. Often more dangerous.
Power (1-β)Probability of correctly detecting a real effectProbability the alarm works when there's a real fire
Tradeoff: Decreasing α (stricter test) reduces Type I errors but increases Type II errors. You can't minimize both without increasing sample size. In medicine, Type II errors are often worse (missing a disease). In spam detection, Type I errors are worse (blocking real emails).

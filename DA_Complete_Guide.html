Subqueries & CTEs PRACTICE

CTE vs Subquery

-- CTE (Common Table Expression) — cleaner, reusable
WITH CustomerRevenue AS (
    SELECT CustomerID, SUM(Amount) AS Revenue
    FROM   Orders
    GROUP BY CustomerID
)
SELECT * FROM CustomerRevenue
WHERE  Revenue > 10000;

-- Correlated Subquery — runs once per outer row
SELECT p.ProductName
FROM   Products p
WHERE  p.UnitPrice > (SELECT AVG(UnitPrice) FROM Products);
Q13 Using a CTE, find customers whose total spend is above the average customer spend. Hard ▼
Answer
WITH CustomerSpend AS (
    SELECT
        c.CustomerID,
        c.CompanyName,
        ROUND(SUM(od.Quantity * od.UnitPrice * (1-od.Discount)), 2) AS TotalSpend
    FROM Customers c
    JOIN Orders o          ON c.CustomerID = o.CustomerID
    JOIN [Order Details] od ON o.OrderID    = od.OrderID
    GROUP BY c.CustomerID, c.CompanyName
)
SELECT *
FROM   CustomerSpend
WHERE  TotalSpend > (SELECT AVG(TotalSpend) FROM CustomerSpend)
ORDER BY TotalSpend DESC;

Q15 Using a multi-CTE query: first calculate monthly revenue, then calculate month-over-month growth %. Hard ▼
Answer
WITH MonthlyRevenue AS (
    SELECT
        strftime('%Y-%m', o.OrderDate) AS Month,
        ROUND(SUM(od.Quantity * od.UnitPrice * (1-od.Discount)), 2) AS Revenue
    FROM Orders o
    JOIN [Order Details] od ON o.OrderID = od.OrderID
    GROUP BY Month
),
WithLag AS (
    SELECT
        Month,
        Revenue,
        LAG(Revenue) OVER (ORDER BY Month) AS PrevRevenue
    FROM MonthlyRevenue
)
SELECT
    Month,
    Revenue,
    PrevRevenue,
    ROUND((Revenue - PrevRevenue) * 100.0 / PrevRevenue, 1) AS GrowthPct
FROM WithLag;

Q16 Rank employees by total sales revenue. Show each employee, their total sales, and their rank. Medium ▼
Answer
SELECT
    e.FirstName || ' ' || e.LastName AS Employee,
    ROUND(SUM(od.Quantity * od.UnitPrice * (1-od.Discount)), 2) AS TotalSales,
    RANK() OVER (ORDER BY SUM(od.Quantity * od.UnitPrice * (1-od.Discount)) DESC) AS SalesRank

JOIN   [Order Details] od ON o.OrderID    = od.OrderID

Q17 For each product, calculate a 3-month moving average of monthly sales quantity. Hard ▼
Answer
WITH MonthlySales AS (
    SELECT
        od.ProductID,
        strftime('%Y-%m', o.OrderDate) AS Month,
        SUM(od.Quantity) AS TotalQty
    FROM [Order Details] od
    JOIN Orders o ON od.OrderID = o.OrderID
    GROUP BY od.ProductID, Month
)
SELECT
    ProductID, Month, TotalQty,
    ROUND(AVG(TotalQty) OVER (
        PARTITION BY ProductID
        ORDER BY Month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ), 1) AS MovingAvg3M
FROM MonthlySales;
Q18 Get the top 2 products by revenue within each category using ROW_NUMBER(). Hard ▼
Answer
WITH ProductRevenue AS (
    SELECT
        p.ProductName,
        p.CategoryID,
        ROUND(SUM(od.Quantity * od.UnitPrice), 2) AS Revenue
    FROM Products p
    JOIN [Order Details] od ON p.ProductID = od.ProductID
    GROUP BY p.ProductID
),
Ranked AS (
    SELECT *,
        ROW_NUMBER() OVER (
            PARTITION BY CategoryID
            ORDER BY Revenue DESC
        ) AS rn
    FROM ProductRevenue
)
SELECT ProductName, CategoryID, Revenue, rn
FROM   Ranked
WHERE  rn <= 2;
Q19 Calculate cumulative (running total) revenue by order date. Show OrderDate, daily revenue, and cumulative revenue. Hard ▼
Answer
WITH DailyRev AS (
    SELECT
        o.OrderDate,
        ROUND(SUM(od.Quantity * od.UnitPrice * (1-od.Discount)), 2) AS DailyRevenue
    FROM Orders o
    JOIN [Order Details] od ON o.OrderID = od.OrderID
    GROUP BY o.OrderDate
)
SELECT
    OrderDate,
    DailyRevenue,
    ROUND(SUM(DailyRevenue) OVER (
        ORDER BY OrderDate
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ), 2) AS CumulativeRevenue
FROM DailyRev;
Advanced SQL & Interview Questions INTERVIEW
Interview Question
What is the difference between WHERE and HAVING?
WHERE filters individual rows before aggregation. HAVING filters grouped results after GROUP BY. You cannot use aggregate functions (SUM, COUNT, etc.) in WHERE, but you can in HAVING. Example: WHERE UnitPrice > 10 filters rows; HAVING COUNT(*) > 5 filters groups.
Interview Question
What is a correlated subquery? When would you use it vs a CTE?
A correlated subquery references columns from the outer query and executes once per outer row (slow on large data). A CTE (WITH clause) is computed once and can be referenced multiple times — better for readability and performance. Use correlated subqueries for "per-row comparisons" (e.g., find products above average in their own category). Use CTEs when you need the result in multiple places or want cleaner code.
Interview Question
Explain ROW_NUMBER vs RANK vs DENSE_RANK with an example.
For scores [100, 90, 90, 80]:
• ROW_NUMBER → 1, 2, 3, 4 (always unique, arbitrary tie-breaking)
• RANK → 1, 2, 2, 4 (ties get same rank, next rank skips)
• DENSE_RANK → 1, 2, 2, 3 (ties get same rank, no skips)
Use ROW_NUMBER for deduplication, RANK/DENSE_RANK for leaderboards.
Q20 Find duplicate orders — same CustomerID, EmployeeID, and OrderDate appearing more than once. Hard ▼
Answer
SELECT CustomerID, EmployeeID, OrderDate, COUNT(*) AS DupeCount
FROM   Orders
GROUP BY CustomerID, EmployeeID, OrderDate
HAVING COUNT(*) > 1;
-- To delete dupes, keep only lowest OrderID:
DELETE FROM Orders
WHERE  OrderID NOT IN (
    SELECT MIN(OrderID)
    FROM   Orders
    GROUP BY CustomerID, EmployeeID, OrderDate
);
Q21 Write a query to pivot quarterly sales by year (Q1, Q2, Q3, Q4 as columns) using CASE WHEN. Hard ▼
Answer
SELECT
    strftime('%Y', OrderDate) AS Year,
    ROUND(SUM(CASE WHEN strftime('%m', OrderDate) IN('01','02','03') THEN Amount ELSE 0 END),2) AS Q1,
    ROUND(SUM(CASE WHEN strftime('%m', OrderDate) IN('04','05','06') THEN Amount ELSE 0 END),2) AS Q2,
    ROUND(SUM(CASE WHEN strftime('%m', OrderDate) IN('07','08','09') THEN Amount ELSE 0 END),2) AS Q3,
    ROUND(SUM(CASE WHEN strftime('%m', OrderDate) IN('10','11','12') THEN Amount ELSE 0 END),2) AS Q4
FROM Orders
GROUP BY Year;

VLOOKUP, INDEX-MATCH, XLOOKUP PRACTICE

The Lookup Evolution

-- VLOOKUP (left column only, approximate/exact)
=VLOOKUP(lookup_value, table_array, col_index_num, [FALSE])

-- INDEX+MATCH (lookup in any direction)
=INDEX(return_range, MATCH(lookup_value, lookup_range, 0))

-- XLOOKUP (modern, most powerful)
=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found])
E6 Employee IDs are in Sheet1!A:A, salaries in Sheet1!C:C. Look up employee "E105" salary from Sheet2 using XLOOKUP. Easy ▼
Answer
=XLOOKUP("E105", Sheet1!A:A, Sheet1!C:C, "Not Found")
-- With a cell reference for the lookup value (say A2 has the ID):
=XLOOKUP(A2, Sheet1!A:A, Sheet1!C:C, "Not Found")
E7 VLOOKUP can only look right. Use INDEX+MATCH to look up a product's category (column B) given its ProductID is in column E (to the right of B). Medium ▼
Answer
-- Assume ProductID in col E, Category in col B (to the LEFT)
-- VLOOKUP can't go left. INDEX+MATCH can:
=INDEX(B:B, MATCH(lookup_id, E:E, 0))
-- MATCH returns the row position; INDEX retrieves the value from col B
💡 This is the main reason INDEX+MATCH beats VLOOKUP — it can look in any direction and the lookup column doesn't need to be first.
E8 Use XLOOKUP to return MULTIPLE columns at once: Given an employee ID in A2, return their Name, Department, and Salary simultaneously in one formula. Hard ▼
Answer
-- Enter in cell C2, it will SPILL into D2 and E2
=XLOOKUP(A2, EmployeeID_col, Name_col:Salary_col)
-- Or explicitly:
=XLOOKUP(A2, B:B, C:E)  -- returns 3 columns
💡 XLOOKUP spills automatically in Excel 365 — one formula returns multiple columns. This is a massive productivity boost over doing 3 separate VLOOKUPs.

Excel Interview Questions INTERVIEW
Interview Question
When would you use SUMPRODUCT vs SUMIFS?
SUMIFS is simpler and faster for straightforward multi-criteria sums. SUMPRODUCT is more flexible — it can handle array operations, partial text matches with wildcards combined with other logic, and calculations that SUMIFS can't do (like conditional weighted averages). SUMPRODUCT also works in older Excel versions that lack SUMIFS.
Interview Question
What's the difference between absolute and relative references? When do you use $A$1 vs A1 vs $A1 vs A$1?
• A1 (relative) — both row and column shift when copied
• $A$1 (absolute) — neither changes; lock a constant like tax rate
• $A1 (mixed, locked column) — column stays A, row shifts; for referencing a whole column table
• A$1 (mixed, locked row) — row stays 1, column shifts; for referencing a header row
Use F4 to toggle between these modes.
Interview Question
How do you handle #N/A errors in VLOOKUP results across a large dataset?
Wrap in IFERROR: =IFERROR(VLOOKUP(...), "Not Found"). Or use IFNA for only #N/A errors (not other error types). Best practice: use XLOOKUP which has a built-in if_not_found parameter and is more robust. Also check: Is the lookup column formatted consistently? Numbers stored as text cause silent mismatches.

CALCULATE — The Most Important DAX Function

-- CALCULATE modifies the filter context
Measure = CALCULATE(expression, filter1, filter2, ...)

-- Example: Sales for only the "North" region
North Sales = CALCULATE(SUM(Sales[Revenue]), Sales[Region] = "North")

-- Example: Remove all filters (total ignoring slicers)
All Sales = CALCULATE(SUM(Sales[Revenue]), ALL(Sales))
P1 Write a DAX measure to calculate Total Revenue, Average Order Value, and Order Count. Easy ▼
Answer
Total Revenue = SUM(Sales[Revenue])

Order Count = COUNTROWS(Orders)
-- Or: DISTINCTCOUNT(Orders[OrderID])

Avg Order Value = DIVIDE([Total Revenue], [Order Count], 0)
-- DIVIDE(numerator, denominator, alternate) — handles /0 safely
P2 Write a measure for % contribution: each product's sales as a % of total sales (should always show % of grand total regardless of filters). Medium ▼
Answer
% of Total Sales = 
DIVIDE(
    SUM(Sales[Revenue]),
    CALCULATE(SUM(Sales[Revenue]), ALL(Products[ProductName])),
    0
)
-- ALL() removes filter on Products so denominator = grand total

Power BI Interview Question
What is the difference between a Measure and a Calculated Column in Power BI?
Calculated Column: Computed row-by-row when data loads. Stored in the model (uses memory). Evaluated in row context. Useful for static values you need to filter/slice on (e.g., category bins, full name = first + last).

Measure: Computed dynamically at query time based on current filter context. NOT stored row-by-row. Uses filter context. Better for aggregations in visuals. Always prefer measures over calculated columns for aggregations — they're more efficient.

Power BI Interview Question
What is cardinality in Power BI relationships, and why does it matter?
Cardinality describes the uniqueness relationship between two tables:
• 1:many — one row in dim table links to many rows in fact table (most common in star schema)
• many:1 — same, just from the other direction
• 1:1 — both tables have unique keys (rare; often you can merge the tables)
• many:many — avoid when possible; requires bridge tables or special handling
Wrong cardinality → wrong filter propagation → wrong report numbers.
More Power BI Interview Questions INTERVIEW
Interview Question
What are Row-Level Security (RLS) and how do you implement it?
RLS restricts data access based on user identity. In Power BI Desktop: Modeling → Manage Roles → Create role with DAX filter. Example: [Region] = USERNAME() or a table-based approach. Then assign users to roles in Power BI Service. Dynamic RLS maps the logged-in user to a security table, so each person only sees their own data automatically.
Interview Question
Import Mode vs DirectQuery — when to use each?
Import Mode: Data is loaded into Power BI's in-memory engine. Fast queries, full DAX support, but data is only as fresh as the last refresh. Best for most analytics use cases.

DirectQuery: Queries sent live to source database on every interaction. Always fresh but slower. Limited DAX functions. Use when: data is too large to import, real-time accuracy is critical, or compliance requires data to stay in source.

Python Interview Questions INTERVIEW
Interview Question
What is the difference between loc and iloc in pandas?
loc is label-based indexing — uses actual index labels and column names. df.loc[5, 'Revenue'] gets the row with index label 5.
iloc is integer-position based — uses 0-based positions. df.iloc[5, 2] gets the 6th row, 3rd column regardless of labels.
When the index is 0-based integers (default), they behave similarly but loc is inclusive on both ends while iloc excludes the end (like Python slicing).

# P(85 <= X <= 115)

print(f"P(85≤X≤115) = {p:.4f}")  # ≈ 0.6827 (68.27%) = ±1σ rule

print(f"90th percentile: {p90:.1f}")  # ≈ 119.2

Hypothesis Testing Framework

StepAction
1. State hypothesesH₀ (null) and H₁ (alternative)
2. Choose significance levelα = 0.05 (95% confidence) or 0.01
3. Select testBased on data type, groups, distribution
4. Calculate test statistic & p-valuet-stat, z-stat, F-stat, chi-square
5. DecisionIf p < α → reject H₀; if p ≥ α → fail to reject H₀
6. ConclusionState in context of the problem
⚠️ p-value is NOT the probability that H₀ is true. It's the probability of getting results as extreme as observed, IF H₀ were true.

S5 A new training program claims to increase sales. 10 salespeople are tested before and after. Before: [45,52,48,61,57,42,59,55,51,58]. After: [52,58,55,65,63,47,64,62,56,63]. Test at α=0.05 whether the program worked. Hard ▼
Answer
from scipy.stats import ttest_rel
import numpy as np

before = np.array([45,52,48,61,57,42,59,55,51,58])
after  = np.array([52,58,55,65,63,47,64,62,56,63])

# H₀: mean difference = 0 (no effect)
# H₁: after > before (one-tailed)
t_stat, p_two = ttest_rel(after, before)
p_one = p_two / 2

print(f"Mean before: {before.mean():.1f}, after: {after.mean():.1f}")
print(f"t-statistic: {t_stat:.3f}")
print(f"p-value (one-tailed): {p_one:.4f}")

if p_one < 0.05:
    print("Reject H₀: Training significantly increased sales")
else:
    print("Fail to reject H₀")
💡 Paired t-test is used here because the SAME people were measured twice. If it were two different groups, use independent t-test.

S7 Explain Type I and Type II errors with a business example. What is statistical power?
Medium ▼
Conceptual Answer
Type I Error (False Positive, α): Rejecting H₀ when it's actually true. Example: Concluding a drug works when it doesn't (α controls this — lower α = stricter)

Type II Error (False Negative, β): Failing to reject H₀ when H₁ is true. Example: Missing a real drug effect (β is the probability of this)

Statistical Power = 1 − β: Probability of correctly detecting a real effect. Increase power by: larger sample size, larger effect size, higher α (trade-off)

There's always a trade-off: stricter significance (lower α) → higher chance of Type II error.

S9 What are the 4 key assumptions of linear regression? How do you check them? Hard ▼
Answer
LINE assumptions:

1. Linearity: Y and X have a linear relationship → Check: scatter plot, residuals vs fitted plot (should show no pattern)

2. Independence: Observations are independent → Check: time-series data may violate this; use Durbin-Watson test

3. Normality of residuals → Check: Q-Q plot (should follow diagonal line), Shapiro-Wilk test

4. Equal Variance (Homoscedasticity): Residuals have constant variance → Check: residuals vs fitted plot (no funnel shape), Breusch-Pagan test

Bonus: No Multicollinearity in multiple regression → Check: VIF < 5-10
S10 What is the Central Limit Theorem and why is it so important for statistics and data analytics? Easy ▼
Answer
Central Limit Theorem (CLT): The distribution of sample means will approach a Normal distribution as sample size increases (n ≥ 30 is the rule of thumb), regardless of the shape of the original population distribution.

Why it matters:
• Allows us to use t-tests and z-tests even when the underlying data isn't perfectly normal
• Justifies constructing confidence intervals using the normal distribution
• Foundation of most hypothesis tests in inferential statistics
• Mean of sample means = population mean (μ), Std Error = σ/√n
Statistics Interview Questions INTERVIEW
Interview Question
What is a confidence interval? If a 95% CI for a difference is (2, 18), what does that mean?
A confidence interval is a range of plausible values for a population parameter, calculated from sample data. "95% CI" means that if we repeated the experiment 100 times, approximately 95 of those intervals would contain the true population parameter.

If the 95% CI for the difference is (2, 18): Since 0 is not in the interval, the difference is statistically significant at α=0.05. We estimate the true difference is between 2 and 18 units, with 95% confidence.
Interview Question
What is the difference between correlation and causation? Give a real-world example of spurious correlation.
Correlation = two variables move together (Pearson r measures linear association, −1 to +1).
Causation = one variable actually causes the other.

Classic spurious correlation: Ice cream sales and drowning deaths both increase in summer. r ≈ 0.8. But ice cream doesn't cause drowning — both are caused by a confounding variable (hot weather → more swimming + more ice cream).

To establish causation: randomized controlled experiments (A/B tests), or causal inference methods like difference-in-differences, instrumental variables.
Interview Question
When would you use median over mean? When would you use mode?
Use Median when: Data is skewed (income, house prices), has outliers, or is ordinal. It's "robust" — 1 extreme value won't move it much.

Use Mean when: Data is roughly symmetric/normal, you need a value for further calculations (CLT, t-tests use mean), or you're computing expected values.

Use Mode when: Data is categorical (most common product sold, most frequent city), or you need the "most likely" value. Also useful for bimodal distributions where mean/median fall between two peaks.
Interview Question
Explain p-hacking and how to avoid it in A/B testing.
P-hacking = running a test until p < 0.05 appears by chance, or testing multiple variations without correcting for multiple comparisons. If you run 20 independent tests at α=0.05, you'd expect ~1 false positive by chance alone.

How to avoid:
• Pre-register your hypothesis and sample size before running the test
• Calculate required sample size upfront using power analysis
• Don't peek at results and stop early
• If testing multiple variants, use Bonferroni correction (α/k) or FDR control
• Set a fixed experiment duration before starting
